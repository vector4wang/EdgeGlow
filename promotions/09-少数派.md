## 正文

### 引言

AI 编程工具正在改变开发者的工作方式。Claude Code、Cursor、GitHub Copilot 等工具让编程效率倍增，但它们都有一个共同的盲区：**你看不到 AI 在做什么**。

当你用 Claude Code 编程时，AI 在后台思考、读文件、执行命令，但你不知道它是否还在工作，还是在等待你的输入。你只能不断切换窗口查看，打断了心流。

今天介绍一个开源小工具 **EdgeGlow**，它用一种优雅的方式解决了这个问题。

---

### 一、EdgeGlow 是什么

EdgeGlow 是一个 macOS 菜单栏小工具，它在屏幕边缘显示流光效果：

- 🟢 **AI 思考中** → 流光沿屏幕边缘旋转流动
- 🔴 **完成/等待** → 流光 1.5s 淡出消失

![EdgeGlow 效果](https://github.com/vector4wang/EdgeGlow/raw/main/images/效果界面.jpg)

你不需要切换窗口，余光一扫就知道 AI 在做什么。

### 二、安装与配置

#### 系统要求

- **macOS**: 13.0 (Ventura) 及以上
- **架构**: 支持 Intel 和 Apple Silicon
- **大小**: 仅 892KB

#### 安装步骤

**方式一：下载 Release（推荐）**

1. 前往 [GitHub Releases](https://github.com/vector4wang/EdgeGlow/releases)
2. 下载 `EdgeGlow-v1.0.0.zip`
3. 解压后双击打开 `EdgeGlow.app`

> ⚠️ 首次打开如遇「无法验证开发者」提示，请 **右键 → 打开** 即可。

**方式二：源码编译**

```bash
git clone https://github.com/vector4wang/EdgeGlow.git
cd EdgeGlow

# 编译（自动构建 Universal Binary: arm64 + x86_64）
./build.sh

# 运行
open Build/EdgeGlow.app
```

#### 配置 Agent 联动

EdgeGlow 通过 HTTP Hook 与 AI Agent 通信。你需要在 Agent 中配置对应的事件触发。

**Claude Code 配置**

1. 点击菜单栏 ✦ 图标 → 设置
2. 点击「配置 Agent Hooks」
3. 选择「Claude Code 查看配置」
4. 点击「复制引导词」
5. 将引导词粘贴到 Claude Code 对话框中发送
6. Agent 会自动帮你完成 `~/.claude/settings.json` 的配置

![设置界面](https://github.com/vector4wang/EdgeGlow/raw/main/images/设置界面.jpg)

**手动配置参考**

如果你更喜欢手动配置，可以在 `~/.claude/settings.json` 中添加：

```json
{
  "hooks": {
    "UserPromptSubmit":  [{"hooks": [{"type": "command", "command": "curl -s http://127.0.0.1:9876/start"}]}],
    "PreToolUse":        [{"hooks": [{"type": "command", "command": "curl -s http://127.0.0.1:9876/start"}]}],
    "PostToolUse":       [{"hooks": [{"type": "command", "command": "curl -s http://127.0.0.1:9876/pulse"}]}],
    "Stop":              [{"hooks": [{"type": "command", "command": "curl -s http://127.0.0.1:9876/stop"}]}]
  }
}
```

**Hermes Agent 配置**

同样通过设置界面获取引导词，发给 Hermes Agent 即可。

### 三、功能详解

![功能界面](https://github.com/vector4wang/EdgeGlow/raw/main/images/功能界面.jpg)

#### 1. 四种颜色主题

EdgeGlow 提供 4 种预设颜色主题，满足不同场景和心情：

| 主题 | 色系 | 适用场景 |
|:----:|:----:|:--------:|
| 🌈 炫酷 | 彩虹全色谱 | 日常使用，视觉冲击最强 |
| 🌊 柔和 | 低饱和蓝紫 | 夜间 / 长时间使用 |
| 🔥 烈焰 | 红橙黄暖色 | 热情满满地写代码 |
| ❄️ 冰雪 | 白蓝冷色 | 冷静思考，专注模式 |

#### 2. 完全可定制

你可以在设置中调整：

- **速度**：流光旋转速度（1-10）
- **光带宽度**：光效粗细（1-10）
- **亮度**：整体亮度（0.3-1.0）
- **旋转方向**：顺时针 / 逆时针

所有设置实时生效，无需重启应用。

#### 3. 多显示器支持

EdgeGlow 自动适配所有屏幕的联合包围区域。当你插拔显示器时，流光会自动重新计算路径（500ms 延迟防抖）。

#### 4. 多终端支持

如果你同时开多个 Claude Code 终端，EdgeGlow 使用引用计数，只有所有终端都完成后流光才会消失。一个终端的 `/stop` 不会影响其他正在工作的终端。

#### 5. 中英双语

EdgeGlow 支持中文和英文界面，会根据系统语言自动切换。

### 四、技术背景

EdgeGlow 是用纯 Swift + SwiftUI 编写的原生 macOS 应用，没有第三方依赖。

#### 核心技术栈

- **NSWindow**: 全屏透明覆盖窗口
- **CALayer + CAShapeLayer**: 四层光效渲染
- **CIFilter**: CIGaussianBlur 高斯模糊
- **Timer**: 60fps 驱动 lineDashPhase 动画
- **NWListener**: HTTP 控制服务器
- **Combine**: 响应式设置管理
- **UserDefaults**: 设置持久化

#### 性能指标

| 指标 | 数值 | 说明 |
|:----:|:----:|:----:|
| CPU 占用 | **~0%** | Timer 60fps 仅更新一个 CGFloat 属性 |
| 内存占用 | **~50MB** | 4 层 CAShapeLayer + CIFilter |
| 启动时间 | **< 0.5s** | 无网络请求，纯本地运行 |
| 网络流量 | **0** | 仅监听 127.0.0.1，不对外通信 |

#### 安全设计

- HTTP 服务仅绑定 `127.0.0.1`，外部无法访问
- 仅接受 `GET` 请求，拒绝 POST/PUT/DELETE
- 无 CORS 头，网页 JavaScript 无法直接调用
- 不收集任何用户数据，无遥测

### 五、使用场景

#### 场景一：Claude Code 编程

这是 EdgeGlow 最主要的使用场景。当你用 Claude Code 写代码时：

1. 你给 AI 一个任务
2. AI 开始工作，屏幕边缘亮起流光
3. 你可以切换到浏览器看文章，或者去倒杯水
4. 余光一扫，看到流光还在，就知道 AI 还在工作
5. 流光消失，就知道 AI 完成了，可以回来继续

#### 场景二：多终端并行

如果你同时开多个 Claude Code 终端处理不同任务，EdgeGlow 会显示所有终端的状态。只有所有终端都完成后流光才会消失。

#### 场景三：Hermes Agent

EdgeGlow 也支持 Hermes Agent，配置方式类似。

---

### 六、注意事项

1. **屏幕录制权限**：EdgeGlow 使用全屏透明覆盖窗口，部分 macOS 版本可能需要在「系统设置 → 隐私与安全 → 屏幕录制」中授权。

2. **全屏应用**：当其他应用处于全屏模式时，EdgeGlow 不会覆盖全屏窗口（这是 macOS 的安全限制）。

3. **Focus / 勿扰模式**：EdgeGlow 不会发送通知，不受勿扰模式影响。

4. **端口冲突**：默认端口 `9876`，如被占用请在设置中更改。

### 七、总结

EdgeGlow 是一个小巧但实用的工具，它用一种优雅的方式解决了 AI 编程中的「看不见」问题。

**优点**：
- ✅ 设计优雅，流光效果美观
- ✅ 轻量高效，几乎不占资源
- ✅ 开源免费，MIT 协议
- ✅ 配置简单，复制引导词发给 AI 即可
- ✅ 安全设计，仅本地通信

**不足**：
- ⚠️ 仅支持 macOS
- ⚠️ 需要配置 Agent Hooks（但引导词让这变得简单）
- ⚠️ 全屏应用无法显示流光（macOS 限制）

如果你也在用 Claude Code 或其他 AI 编程工具，EdgeGlow 值得一试。
