## 专栏文章正文

### 引言

当我们谈论 AI 编程工具时，我们关注的是模型的能力：代码生成、上下文理解、多文件协同。但有一个被忽视的维度：**开发环境的可视化反馈**。

这篇文章，我想从一个具体的小工具 EdgeGlow 说起，聊聊 AI 编程时代的「环境感知」设计。

---

### 一、AI 编程的痛点：你看不见 AI 在做什么

用 Claude Code 写代码时，你会遇到这样的场景：

1. 你给 AI 一个任务，说"帮我重构这个模块"
2. AI 开始工作：读文件、思考、写代码、执行命令
3. 你切换到浏览器看文章，或者去倒杯水
4. 回来时，你不知道 AI 是还在工作，还是已经完成了在等你操作
5. 你 alt-tab 切过去看，发现 AI 确实在等你

**这个「看不见」的问题，打断了你的心流。**

传统的 CLI 工具有明确的输出：进度条、日志、完成提示。但 AI Agent 是异步的、长时间运行的，传统的 CLI 反馈模式不再适用。

---

### 二、解决方案：从「焦点切换」到「余光感知」

人类有两种视觉注意力：

- **焦点视觉**：你正在看的东西（比如浏览器里的文章）
- **余光视觉**：你余光里能看到的东西（比如屏幕边缘的变化）

传统的做法是 alt-tab 切换窗口，这是**焦点切换**，成本高，打断心流。

更好的做法是利用**余光感知**：在屏幕边缘显示一个微妙的视觉信号，你不需要切换窗口，余光一扫就知道 AI 的状态。

**这就是 EdgeGlow 的设计理念。**

---

### 三、EdgeGlow：让 AI 的工作状态「可见」

EdgeGlow 是一个 macOS 菜单栏小工具，它在屏幕边缘显示流光效果：

- 🟢 **AI 思考中** → 流光沿屏幕边缘旋转流动
- 🔴 **完成/等待** → 流光 1.5s 淡出消失

![EdgeGlow 效果](https://github.com/vector4wang/EdgeGlow/raw/main/images/效果界面.jpg)

你不需要切换窗口，余光一扫就知道 AI 在做什么。

### 四、技术实现：四个值得聊的设计决策

EdgeGlow 的代码量不大，但有几个技术决策值得聊聊。

#### 1. 四层光效模拟霓虹灯

为了做出真实的霓虹灯效果，我用了 4 层 `CAShapeLayer` 叠加：

```
Layer 1: 宽线 + 高斯模糊(12) + 低透明度 → 外层光晕
Layer 2: 中线 + 模糊(8)  + 中透明度 → 中层光晕  
Layer 3: 细线 + 模糊(2)  + 高透明度 → 主线
Layer 4: 最细 + 无模糊    + 全透明度 → 亮芯
```

每层独立运行动画，四层叠加 = 真实的霓虹灯效果。

这个技巧在 UI 设计中很常见：真实的发光效果不是单层能做到的，需要多层叠加，外层模糊扩散，内层锐利明亮。

#### 2. Timer 驱动 vs CABasicAnimation

一开始我用 `CABasicAnimation` 做 `lineDashPhase` 动画，但发现窗口隐藏再显示后，动画会卡住甚至反转。

**根本原因**：`CABasicAnimation` 依赖 Core Animation 的动画系统，窗口隐藏时动画状态会丢失。

**解决方案**：改用 `Timer` 在 60fps 下直接更新 `lineDashPhase`。

```swift
flowTimer = Timer.scheduledTimer(withTimeInterval: 1.0/60.0, repeats: true) {
    dashPhase += speed * dt
    sublayers.forEach { $0.lineDashPhase = dashPhase }
}
```

这不依赖 Core Animation 的动画系统，窗口怎么切换都不会出问题。

**设计哲学**：当系统提供的抽象不够可靠时，退到底层，自己控制状态。

#### 3. 引用计数支持多终端

如果你同时开多个 Claude Code 终端，一个 `/stop` 不应该灭掉流光 —— 其他终端还在工作。

解决方案：引用计数。

```swift
case "/start": activeCount += 1
case "/stop":  activeCount -= 1
if activeCount == 0 { hide() }
```

还有 60 秒安全超时：如果 Agent 崩溃没发 `/stop`，60 秒后自动归零。

**设计哲学**：考虑边界情况，但不让正常流程变复杂。

#### 4. 安全优先的 HTTP 服务

EdgeGlow 在 `127.0.0.1:9876` 运行 HTTP 服务，我做了几个安全设计：

- `acceptLocalOnly = true`：仅接受本地连接
- 仅接受 GET 请求，拒绝 POST/PUT/DELETE
- 无 CORS 头，网页 JavaScript 无法调用
- 不收集任何数据

**设计哲学**：本地工具也要考虑安全，尤其是当它运行在你的电脑上，监听网络端口时。

### 五、更深的思考：AI 工具需要「环境感知」设计

EdgeGlow 是一个小工具，但它代表了一个更大的趋势：**AI 工具需要「环境感知」设计**。

传统的 CLI 工具是同步的、短时间运行的，反馈模式是日志、进度条。

AI Agent 是异步的、长时间运行的，需要新的反馈模式：

- **余光感知**：屏幕边缘的微妙信号（EdgeGlow）
- **声音反馈**：完成任务时的提示音
- **触觉反馈**：智能手表的震动
- **环境光**：智能灯泡的颜色变化

这些反馈的共同点：**不打断你的焦点，但让你感知到 AI 的状态**。
