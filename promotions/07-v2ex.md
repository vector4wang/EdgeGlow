## 正文

各位 V 友好，

周末做了个小工具 —— **EdgeGlow**。

### 起因

用 Claude Code 写代码时，AI 在后台思考，你看不见它是否在忙。EdgeGlow 在屏幕边缘显示流光效果：AI 工作时流光旋转，完成后淡出。

就是这样，很简单。

### 效果

![EdgeGlow 效果](https://github.com/vector4wang/EdgeGlow/raw/main/images/效果界面.jpg)

- 🟢 AI 思考中 → 流光沿屏幕边缘旋转流动
- 🔴 完成/等待 → 流光 1.5s 淡出消失

### 技术栈

**纯 Swift + SwiftUI，无第三方依赖**

- CAShapeLayer + CIGaussianBlur 四层光效模拟霓虹灯
- Timer 驱动 lineDashPhase 动画（不依赖 Core Animation，更可靠）
- NWListener HTTP 服务器（仅 127.0.0.1）
- 引用计数支持多终端同时工作

### 特性

- 4 种颜色主题（炫酷、柔和、烈焰、冰雪）
- 多显示器支持（自动适配屏幕变化）
- 完全可定制（速度、宽度、亮度、方向）
- 中英双语
- ~0% CPU, ~50MB 内存
- 只有 892KB
- Universal Binary（arm64 + x86_64）

### 配置方式

打开设置 → 配置 Agent Hooks → 复制引导词 → 发给 Claude Code → 自动配置完成。

或者手动在 `~/.claude/settings.json` 添加 hooks 配置。

### 安全考虑

- HTTP 服务仅绑定 127.0.0.1（`acceptLocalOnly`）
- 仅接受 GET 请求
- 无 CORS 头，网页 JS 无法调用
- 不收集任何数据

## 互动准备

### 可能的回复 & 回答

**Q: 这个和 XXX 有什么区别？**
A: EdgeGlow 专注于 AI 编程场景，通过 HTTP hooks 和 Claude Code 联动。其他工具可能是通用的屏幕特效工具。

**Q: 为什么不用 Electron？**
A: 太重了。SwiftUI 原生应用，892KB 搞定，Electron 至少 100MB+。

**Q: 支持 Windows 吗？**
A: 目前只支持 macOS，因为用了 Core Animation。Windows/Linux 版本需要其他开发者贡献。

**Q: 能不能支持其他 AI 工具？**
A: 可以。只要工具能发 HTTP 请求，调用 localhost:9876/start 和 /stop 就行。

**Q: 代码质量怎么样？**
A: 第一个 macOS 应用，代码质量一般。欢迎 PR 和 Issue。

**Q: 为什么用 Timer 不用 CABasicAnimation？**
A: CABasicAnimation 在窗口隐藏再显示后会丢状态，动画会卡住或反转。Timer 驱动更可靠。
