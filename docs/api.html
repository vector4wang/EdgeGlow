1. 概述

协议：HTTP/1.1

方法：仅支持 GET / OPTIONS

地址：http://127.0.0.1:{port}（默认端口 9876，可配置 1024–65535）

绑定：仅本地回环（acceptLocalOnly = true），不接受外部连接

数据格式：请求参数通过 URL Query String 传递；响应为纯文本或 JSON

CORS：允许所有来源（Access-Control-Allow-Origin: *）

EdgeGlow 采用 PID 追踪机制：每个终端窗口通过 ?pid=$PPID 标识自己，支持多终端并行工作。当所有活跃终端都结束后，流光才会关闭。

2. 接口列表

GET /start?pid={pid}

功能：开启屏幕边缘流光效果。注册 PID 到活跃终端集合，启动流光动画。

参数类型必填说明
pidint是终端进程 PID，用于多窗口追踪，推荐使用 $PPID

响应：

HTTP/1.1 200 OK
Content-Length: 2
Connection: close

ok

副作用：PID 加入 activePIDs 集合；启动 120 秒安全超时定时器（超时自动移除 PID）。

GET /stop?pid={pid}

功能：移除 PID，若无其他活跃终端则关闭流光。

参数类型必填说明
pidint是要移除的终端进程 PID

响应：

HTTP/1.1 200 OK
ok

副作用：PID 从 activePIDs 移除，取消安全超时定时器。仅当 activePIDs 为空时调用 onStop 关闭流光。

GET /pulse?pid={pid}

功能：脉冲效果 — 移除 PID，若所有终端都已完成则触发一次脉冲闪烁后淡出。

参数类型必填说明
pidint是要移除的终端进程 PID

响应：

HTTP/1.1 200 OK
ok

副作用：PID 移除 + 取消超时定时器。activePIDs 为空时调用 onPulse（脉冲 → 1.5s 淡出）。

GET /status

功能：查询当前状态，返回活跃终端数量和 PID 列表。

参数：无

响应：

HTTP/1.1 200 OK
Content-Length: 45
Connection: close

{"active_count":2,"pids":[12345,12346]}
字段类型说明
active_countint当前活跃终端数量
pidsint[]活跃终端 PID 列表（升序）
GET /start_legacy deprecated

功能：开启流光（无 PID 追踪，向后兼容旧版 hook 脚本）。

参数：无

响应：200 OK

建议迁移到带 ?pid= 的新接口，以支持多终端并行工作。
GET /stop_legacy deprecated

功能：关闭流光（无 PID 追踪）。

参数：无

响应：200 OK

OPTIONS /*

功能：CORS 预检请求。所有路径均支持。

响应：

HTTP/1.1 204 No Content
Access-Control-Allow-Methods: GET
Access-Control-Allow-Origin: *
Connection: close

3. Hooks 集成

3.1 Claude Code

在 ~/.claude/settings.json 中配置：

{
  "hooks": {
    "UserPromptSubmit": [{
      "hooks": [{"type": "command", "command": "curl -s \"http://127.0.0.1:9876/start?pid=$PPID\""}]
    }],
    "PreToolUse": [{
      "hooks": [{"type": "command", "command": "curl -s \"http://127.0.0.1:9876/start?pid=$PPID\""}]
    }],
    "PostToolUse": [{
      "hooks": [{"type": "command", "command": "curl -s \"http://127.0.0.1:9876/pulse?pid=$PPID\""}]
    }],
    "Stop": [{
      "hooks": [{"type": "command", "command": "curl -s \"http://127.0.0.1:9876/pulse?pid=$PPID\""}]
    }]
  }
}

3.2 Hermes Agent

在 ~/.hermes/agent-hooks/ 下创建脚本：

# pre_llm_call.sh
#!/bin/bash
curl -s "http://127.0.0.1:9876/start?pid=$PPID"

# on_llm_response.sh
#!/bin/bash
curl -s "http://127.0.0.1:9876/pulse?pid=$PPID"

3.3 Hook 触发时机

Hook 事件API 调用效果
UserPromptSubmitGET /start?pid=$PPID用户输入时 → 开启流光
PreToolUseGET /start?pid=$PPIDAI 调用工具前 → 保持流光
PostToolUseGET /pulse?pid=$PPID工具调用完成 → 脉冲后淡出
StopGET /pulse?pid=$PPIDAgent 停止 → 脉冲后淡出

4. 状态机

⚫ 空闲
activePIDs = []
→ /start →
🔴 思考中
activePIDs = [p1, p2, ...]
→ /pulse 或 /stop →
⚫ 空闲
activePIDs = []

多终端并行：每个 /start 增加一个 PID，每个 /pulse 或 /stop 移除一个 PID。
仅当 所有 PID 都被移除 后，流光才会关闭。

安全兜底：每个 PID 注册后启动 120 秒超时定时器，超时自动移除，防止僵尸 PID 导致流光常亮。

5. 错误码

状态码含义触发条件
200 OK成功正常请求处理完成
204 No Content预检通过OPTIONS 请求
404 Not Found路径不存在请求了未定义的路径
405 Method Not Allowed方法不允许使用了 GET/OPTIONS 以外的方法

6. 调用示例

6.1 基本调用

# 开启流光（终端 A）
curl -s "http://127.0.0.1:9876/start?pid=$$"

# 开启流光（终端 B，并行）
curl -s "http://127.0.0.1:9876/start?pid=$$"

# 终端 A 完成 → 脉冲淡出
curl -s "http://127.0.0.1:9876/pulse?pid=12345"

# 查询当前状态
curl -s "http://127.0.0.1:9876/status"
# → {"active_count":1,"pids":[12346]}

# 终端 B 完成 → 脉冲淡出（流光关闭）
curl -s "http://127.0.0.1:9876/pulse?pid=12346"

6.2 自定义端口

# 在设置中修改端口为 8888 后：
curl -s "http://127.0.0.1:8888/start?pid=$PPID"
curl -s "http://127.0.0.1:8888/pulse?pid=$PPID"
curl -s "http://127.0.0.1:8888/status"

6.3 手动控制（不依赖 Hooks）

# 长时任务前后手动控光
curl -s "http://127.0.0.1:9876/start?pid=$$"
sleep 10  # 模拟耗时操作
curl -s "http://127.0.0.1:9876/pulse?pid=$$"
